# Customer-Churn-Prediction-using-ML

📌 Project Overview

Customer churn is one of the biggest challenges faced by subscription-based businesses and telecom companies. In this project, I built a complete Machine Learning pipeline to predict whether a customer is likely to leave the company based on customer behavior, services, and account information.

This project focuses on:
- Data Cleaning & Preprocessing
- Feature Engineering
- Building ML Pipelines
- Model Training & Evaluation
- Hyperparameter Tuning
- Performance Analysis

🧠 Problem Statement

The goal of this project is to identify customers who are likely to churn so companies can take proactive actions to improve customer retention.

Predicting churn helps businesses:
- Reduce customer loss
- Improve customer satisfaction
- Increase revenue
- Build better marketing strategies

📂 Dataset Used

Dataset: Telco Customer Churn Dataset
The dataset contains customer information such as:
- Gender
- Senior Citizen Status
- Internet Services
- Contract Type
- Payment Method
- Monthly Charges
- Total Charges
- Tenure
- Churn Status

🔄 Machine Learning Workflow

- 1️⃣ Data Preprocessing
Handled missing values
Performed categorical encoding using OneHotEncoder
Standardized numerical features using StandardScaler
Built preprocessing pipelines using ColumnTransformer
- 2️⃣ Model Building
Implemented and compared multiple Machine Learning models:
Logistic Regression
Random Forest Classifier
- 3️⃣ Model Evaluation
Used multiple evaluation metrics:
Accuracy Score
Classification Report
Confusion Matrix
ROC-AUC Score
- 4️⃣ Hyperparameter Tuning
Applied GridSearchCV to improve model performance and select the best parameters.
